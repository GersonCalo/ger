## Notas Técnicas

- **Auth fixture**: Cada test crea un usuario único vía API (`/api/v1/auth/register`) y seedea 1 transacción de prueba. No depende de datos previos.
- **Swipe helpers**: Centralizados en `e2e/helpers/swipe.ts` con coordenadas calculadas por `boundingBox`.
- **Retries**: 1 retry en local, 2 en CI (`playwright.config.ts`).
- **Evidencia**: Screenshots y videos solo en fallos (`screenshot: only-on-failure`, `video: retain-on-failure`).
- **Reportes**: HTML (`playwright-report/index.html`), JSON (`playwright-report/results.json`), JUnit (`playwright-report/junit.xml`).
